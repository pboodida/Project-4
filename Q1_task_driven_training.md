## 1. Project Overview and Key Components

### Repository Analysis Summary

AutoGaze (Autoregressive Gazing) is a CVPR 2026 accepted work from NVIDIA, UC Berkeley, and MIT. The system addresses a fundamental inefficiency in modern video understanding pipelines: Vision Transformers (ViTs) and Multimodal LLMs process every patch of every video frame equally, regardless of whether a patch carries new temporal or spatial information. AutoGaze is a lightweight (~3M parameter) pre-ViT module that autoregressively selects the minimal informative patch subset per frame before handing off to any downstream ViT or LLM. It achieves 4×–100× token reduction and up to 19× end-to-end speedup on downstream encoders.

The codebase is structured as a clean four-component modular system:

| Module | Path | Role |
|---|---|---|
| **Gaze Model** | `autogaze/models/autogaze/` | Autoregressive decoder that predicts patch indices |
| **Tasks** | `autogaze/tasks/video_mae_reconstruction/` | Defines reward, loss, and metrics for training |
| **Algorithms** | `autogaze/algorithms/grpo.py`, `ntp.py` | Training logic: GRPO (RL) and NTP (supervised) |
| **Vision Encoders** | `autogaze/vision_encoders/siglip/` | Adapter layer making SigLIP patch-selection-aware |

Training proceeds in two stages: **Stage 1** uses Next Token Prediction (NTP) to learn from pre-computed greedy-search gazing sequences stored in `gazing_labels.json`. **Stage 2** applies Group Relative Policy Optimization (GRPO) with reconstruction reward to discover gazing sequences that outperform the greedy labels. The trained model is deployed by passing `gazing_info` from AutoGaze into the modified ViT's `gazing_info` parameter, which routes only the selected patches through the encoder.

## 2. Deep Reasoning Questions & Analysis

### Question 1

> **AutoGaze uses a task-driven training approach: the gaze model learns which patches to select by optimizing for downstream task performance (e.g., video reconstruction), not by predicting human gaze directly. Why is this indirect training strategy superior to supervised gaze prediction?**

---

### Direct Answer

The task-driven approach is superior because it defines patch informativeness in terms of the actual downstream objective (reconstruction fidelity or model accuracy) rather than in terms of human visual fixation patterns. Human gaze is shaped by cognitive and attentional biases—faces, motion, text—that do not necessarily correspond to which patches are computationally sufficient for accurate video reconstruction or classification. The task-driven approach generates its own ground truth algorithmically, scales to any volume of video without human annotators, and supports reinforcement learning post-training to surpass even the quality of the greedy labels it starts from.

---

### How to Know the Answer — Repository Evidence

**File:** `TRAIN.md` — Stage 1 configuration  
The NTP pre-training command does not pass any human gaze dataset. Ground truth comes from `gazing_labels.json`, which is generated by greedy reconstruction-loss minimization:

```bash
# From TRAIN.md — Stage 1 NTP training
torchrun \
    --nnodes=1 --nproc_per_node=8 \
    -m autogaze.train \
        --config-name video_folder_video_mae_reconstruction_ar_gaze_ntp \
        dataset.gt_gazing_pos_paths.train='<path to gazing_labels.json>' \
        task.recon_model=facebook/vit-mae-large \
        task.recon_model_config.loss_type=l1+dinov2_reg+siglip2 \
        task.recon_model_config.loss_weights=1+0.3+0.3 \
        algorithm.optimize_task_loss_prediction=True \
        trainer.train_gaze=True \
        trainer.train_task=False
```

**Key observation:** `task.recon_model_config.loss_type=l1+dinov2_reg+siglip2` — The supervision signal is pixel-level L1 loss combined with DINOv2 and SigLIP2 feature reconstruction losses. There is zero reference to any human eye-tracking annotation file in the entire repository.

**File:** `TRAIN.md` — Stage 2 GRPO configuration:

```bash
# Stage 2 RL training — reward is reconstruction loss, not human fixation
algorithm.group_size=12 \
algorithm.discount_factor=0.995 \
algorithm.optimize_task_loss_prediction=True \
trainer.train_task=False
```

The `train_task=False` flag confirms the VideoMAE reconstruction model is frozen. AutoGaze is optimized purely against the reconstruction reward signal, not against any externally provided gaze label.

**File:** `TRAIN.md` — Explanation of `gazing_labels.json`:
> *"The `gazing_labels.json` file contains pre-computed ground-truth gazing sequences used in Stage 1 (NTP pre-training)."*

These labels are generated algorithmically, not by human observers.

**File:** `README.md` — Code structure:
> *"tasks: Definition of the task serving as the training objective for gaze models, such as video mae reconstruction. Here everything that's related to the task is defined, including the task model (i.e., the model used in the task such as VideoMAE), the loss of the task (e.g., MAE reconstruction loss), or the reward function used to train the gaze model (e.g., reconstruction reward)"*

### Proof

**Proof 1 — Greedy label generation = task-optimized, not human-annotated**

The training data pipeline (described in `TRAIN.md`) requires downloading `gazing_labels.json` from HuggingFace (`bfshi/AutoGaze-Training-Data`). This file contains pre-computed patch selection sequences found by iteratively selecting the patch that yields the greatest decrease in VideoMAE reconstruction loss at each step. No human subjects are involved. The loss signal is:

```
L_recon = 1.0 × L1_pixel + 0.3 × DINOv2_features + 0.3 × SigLIP2_features
```

This is a pure computational metric. Human gaze prediction datasets (e.g., SALICON, MIT1003) are never referenced in any config or data loading file in the repository.

**Proof 2 — RL stage surpasses greedy labels**

The Stage 2 GRPO training explicitly states its purpose is to discover *better* gazing than Stage 1's greedy labels:

From `TRAIN.md`:
> *"Stage 2: Post-training with Reinforcement Learning (RL) — AutoGaze is further trained with GRPO using reconstruction reward to discover better gazing sequences."*

If the NTP labels (greedy search) were optimal, Stage 2 would provide no benefit. The fact that Stage 2 exists and improves performance proves that the task-driven signal enables self-improvement beyond any fixed label quality ceiling—which is impossible with static human gaze supervision.

**Proof 3 — The reconstruction objective generalizes to diverse downstream tasks**

From `QUICK_START.md`, AutoGaze is deployed with:
```python
gaze_outputs = autogaze_model(
    {"video": video_input_autogaze}, 
    gazing_ratio=0.75, 
    task_loss_requirement=0.7
)
```

This same model, trained purely on reconstruction reward, is then applied to NVILA-8B-HD-Video for QA tasks. The reconstruction-trained patch selector generalizes because informative patches for reconstruction (high temporal change, fine-grained detail) are also informative patches for semantic understanding tasks.

---

### Problem Design

AutoGaze frames the patch selection problem as: *"given the video so far, which single patch, if added to the current selection set, most reduces the reconstruction loss?"* This greedy oracle defines the NTP training labels. The RL stage then optimizes the policy to go beyond single-step greedy to globally better trajectories.

The task-driven formulation solves four problems that human gaze supervision cannot:

1. **Label scalability**: Millions of videos can be labeled algorithmically in parallel
2. **Label consistency**: Reconstruction loss is deterministic; human fixation has high inter-subject variance
3. **Closed-loop improvement**: RL reward allows the model to improve past label quality
4. **Task alignment**: The training signal directly matches inference requirements

---

### Alternative Suggestions

A direct competitor to task-driven training is **saliency map supervision** from models like CLIP or DINO, which produce attention-based pseudo-labels without human annotation. The weakness compared to AutoGaze's approach is that saliency maps are post-hoc explanations of model decisions, not direct measures of patch reconstruction utility. Another alternative is **optical flow-guided selection**, which selects patches with high motion. This is cheaper but misses texture-rich static patches that are critical for reconstruction fidelity.

---

### Summary

AutoGaze's task-driven training is superior to supervised gaze prediction because: (1) the training signal is directly aligned with the deployment objective, (2) labels can be generated algorithmically at any scale without human annotators, (3) RL post-training allows discovery of gazing strategies superior to any fixed ground truth, and (4) reconstruction-based saliency generalizes to diverse downstream tasks without task-specific fine-tuning.

---

## 3. Findings and Conclusion

The task-driven training strategy is the central design choice that makes AutoGaze practical and effective. By defining patch informativeness through reconstruction loss rather than human fixation, AutoGaze avoids the annotation bottleneck of supervised gaze datasets, achieves task-aligned patch selection, and enables continuous improvement through reinforcement learning. Every component in the training pipeline—from `gazing_labels.json` generation to the GRPO reward function in `autogaze/algorithms/grpo.py`—reflects this philosophy consistently.
